# JA Core

Respond in Korean by default.

Use this skill as the common Japanese processing layer.
It does not decide translation direction.
It processes the Japanese sentence supplied by `ko-ja`, `ja-ko`, or the user.

## Output Block

For each Japanese sentence, produce this block:

1. 후리가나형
2. 단어 분석
3. 개별 한자 분석
4. 저장 기록

Write furigana as `漢字(かんじ)`.
Inside `후리가나형`, include both:

- `문장`: the sentence with furigana.
- `전체 읽기`: the full sentence reading entirely in hiragana.

Do not create a separate top-level `히라가나 읽기` section.

## Word Analysis

Analyze by word unit first.

For each important word with kanji, include:

- surface form
- reading in this sentence
- Korean meaning
- kanji contribution inside the word

Include particles when they are a lesson point or useful for frequency tracking.
Do not turn every character into trivia.

## Individual Kanji Analysis

After word analysis, add a separate `개별 한자 분석` section.

For each unique kanji character in the Japanese sentence, include:

- kanji character
- hiragana reading used in this sentence or word
- core Korean meaning
- source word where it appeared

Use the reading from the actual word when possible.
If a kanji has multiple readings but only one appears in the sentence, show only the sentence reading.
If the character is in an ateji, proper noun, or idiomatic reading, say that the reading is word-specific.

## Sentence Save Rules

Use `sentences[]` for Japanese sentences.

- Reuse an existing entry when `text` or `variants` match.
- Otherwise add `sentences[]` with id `jp-s-YYYYMMDD-NNN`.
- Add today's date to `dates` if missing.
- Preserve or add `furigana`, `reading`, and Korean meaning fields.
- Add source paths when relevant.

## Translation Pair Save Rules

Use `translation_pairs[]` only when a Korean source sentence produced the Japanese sentence.

- Reuse an existing pair when `source_text` and `japanese_text` match.
- Otherwise add `translation_pairs[]` with id `jp-t-YYYYMMDD-NNN`.
- Store `source_language: ko`.
- Store `source_text`, `japanese_text`, `furigana`, `reading`, `korean_back_translation`, `tone`, `first_seen`, `dates`, `sentence_id`, and `sources`.
- Add today's date to `dates` if missing.

## Word Save Rules

Use `words[]` for important words and tracked particles in the Japanese sentence.

- Use the lemma or stable dictionary form as `key`.
- Increment `count` for each analyzed occurrence.
- Add today's date to `dates` once per date.
- Add the sentence id to `sentence_ids`.
- Keep `kanji` as character-level components inside each word.
- Each `kanji[]` item must include `form`, `reading`, and `meaning` when practical.
- Use hiragana for `kanji[].reading`.

## Example

Japanese sentence:

```text
日本語を勉強するのが好きです。
```

Core output:

```text
1. 후리가나형
- 문장: 日本語(にほんご)を勉強(べんきょう)するのが好(す)きです。
- 전체 읽기: にほんごをべんきょうするのがすきです

2. 단어 분석
- 日本語(にほんご): 일본어. 日本 = 일본, 語 = 언어.
- 勉強(べんきょう): 공부. 勉 = 힘쓰다, 強 = 강하다.
- 好き(すき): 좋아함.

3. 개별 한자 분석
- 日(に): 해, 날. `日本語` 안에서는 に로 읽는다.
- 本(ほん): 근본, 책. `日本語` 안에서는 ほん으로 읽는다.
- 語(ご): 말, 언어. `日本語` 안에서는 ご로 읽는다.
- 勉(べん): 힘쓰다. `勉強` 안에서는 べん으로 읽는다.
- 強(きょう): 강하다. `勉強` 안에서는 きょう으로 읽는다.
- 好(す): 좋아하다. `好き` 안에서는 す로 읽는다.
```
